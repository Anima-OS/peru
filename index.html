buildinspace | peru

fetch code for your projects and tools, from JavaScript to VimL

Peru is a flexible command line tool for fetching other people's code and merging it into your project. Unlike many other tools, peru is generic, recursive, and doesn't get in the way of your other tools or workflow.

> configurable

Peru consumes a simple YAML file that describes what to fetch, how to fetch it, and where to put it. Normally, this file is called peru.yaml and is checked in with your project. Peru can fetch specific revisions, tags, and branches, and can even hash non-version-controlled files and poll for updates (see the reup command).

For example, if you wanted to fetch a project foo from GitHub at tag 0.1.0, you could use the following YAML.

imports:
  foo: third-party/ # Place in third-party/.

git module foo: # Fetch from git.
  url: https://github.com/example/foo.git
  rev: 0.1.0

The YAML can also define reusable rules, which apply fields to arbitrary modules by chaining or piping them together.

> portable

Peru is written in Python 3 and runs everywhere Python does, including Linux, Mac OS, and Windows (see packages).

Powered by pathlib and shutil, file operations are abstracted by peru and exposed as fields in YAML, so moving and copying files Just Works™ on every platform.

Of course, peru is available via pip and is hosted on PyPI, so installation is easy.

$ pip install peru

> fast

Built atop Python 3's asyncio, peru operates on modules in parallel to fetch code effeciently. Coupled with aggressive caching, peru sync is fast and avoids unnecessary network traffic.

For constrained environments, parallelism can be controlled or completely disabled. The cache can be placed in a custom directory, and can be purged at any time.

> extensible

Out of the box, peru supports many popular and practical module types, including git, hg, curl, and cp. However, projects may need support for something not already in this toolbox.

Peru interacts with module code via a simple extension interface, shelling out to scripts to fetch and update files. Even peru's core modules are built this way, and it's trivial to extend peru with new module types as needed.

Extensions are simple scripts that can be written with any language, like Bash, Python, or C.
